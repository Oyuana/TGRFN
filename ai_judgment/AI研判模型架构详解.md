# AI 智能研判模型架构详解

> 本文档面向竞赛作品报告，完整阐述系统中 **「AI 智能研判」** 这一部分的模型架构、
> 算法原理与工程实现。它衔接在 TGRFN 检测模型（见报告 §2.2）之后，对应研究目标第 4 条
> 「自动应对与研判报告」，并正面回应报告 §1.2.4 中提出的**大模型知识幻觉**难题。
>
> **一句话定位**：TGRFN 负责"判真伪并给出可解释的中间证据"，AI 研判层负责"把这些证据
> 校准成量化研判指标，再由大语言模型翻译成结构化研判报告与处置建议"。
> 二者职责严格分离——**深度模型算数、大模型说话，大模型不参与任何数值裁决**。

---

## 1. 设计动机：为什么在 TGRFN 之上还要做 AI 研判层

TGRFN 模型（报告 §2.2）已能输出"真/假标签 + 置信度 + 关系增强后的传播图谱"。
但在面向社交平台管理人员的真实研判场景中，仅有一个分类结论是不够的，存在三个能力缺口：

| 现实痛点 | 仅有 TGRFN 分类结论时的不足 | AI 研判层的回应 |
|----------|------------------------------|------------------|
| **结论不可解释** | 一个 0.91 的概率无法向审核员/上级解释"凭什么" | 把模型**中间表征**拆成结构冲突、模态贡献、传播规模等**可核查量化指标** |
| **类别不平衡误判** | 真假样本比例不均时，0.5 阈值会系统性偏移（报告 §3.2 已指出不平衡问题） | 用 `odds 与训练集正负样本比` 的比较做**校准判别** |
| **早期/稀疏过自信** | 冷启动阶段（报告 §2.3）传播数据极少，模型易过自信 | 检测稀疏传播并**主动下调置信度**，匹配早期预警的审慎要求 |
| **缺乏自动报告与处置** | 人工撰写研判报告慢且不一致 | 引入 LLM 自动生成**结论/证据/传播分析/处置建议**并一键导出 |

**与报告 §1.2.4 的呼应（反幻觉设计的理论依据）**：报告综述指出，大模型在事实性研判任务上
始终面临**知识幻觉**问题。为彻底规避幻觉风险，本系统**不让大模型直接判别新闻真伪**，
而是把它降级为"解释器"——大模型只能在 TGRFN 给定的、已量化的事实证据范围内组织语言，
**不得重算、不得推翻、不得编造**。这与 PHPFND[3] 等工作"约束/利用幻觉"的思路一致：
我们通过结构化的只读输入，从源头上把幻觉发生的空间压到最小。

## 2. 总体架构

AI 研判层在逻辑上由**四个阶段**串联而成，前后阶段之间只通过标准化数据结构耦合：

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 阶段一：TGRFN 研判中间量提取   (model.py · forward(return_internals)) │
│   复用互注意力协同检测模块(§2.2.4)产出的内容对齐表示 align_c 与        │
│   传播图对齐表示 align_g，及多模态表示能量、传播邻居规模、判别概率。   │
│   产物：ModelInternals                                                 │
└───────────────────────────────┬──────────────────────────────────────┘
                                 ▼
┌──────────────────────────────────────────────────────────────────────┐
│ 阶段二：量化研判内核   (internals.py · 纯函数, 零深度学习依赖)         │
│   把中间量经一组有物理意义的规则映射为可解释的量化研判指标。           │
│   产物：JudgmentMetrics（判别间隔/假新闻风险/结构冲突/模态注意力/     │
│         传播稀疏/预分级/置信度/触发规则）                              │
└───────────────────────────────┬──────────────────────────────────────┘
                                 ▼
┌──────────────────────────────────────────────────────────────────────┐
│ 阶段三：大语言模型解释层   (prompts.py + llm_report.py)               │
│   以量化指标为"只读事实"，由 LLM 翻译成结构化中文研判报告；           │
│   失败自动降级为规则报告(mock)，保证输出结构恒定。                     │
│   产物：研判报告 JSON（Risk_Level/Conclusion/Evidence/                │
│         Propagation_Analysis/Recommendation）                          │
└───────────────────────────────┬──────────────────────────────────────┘
                                 ▼
┌──────────────────────────────────────────────────────────────────────┐
│ 阶段四：服务接口与前端展示   (api.py + 前端.html)                     │
│   FastAPI 以 { metrics, report } 双段返回；前端渲染研判等级、传播      │
│   拓扑演化、详细报告，并一键导出 PDF 研判报告。                        │
└──────────────────────────────────────────────────────────────────────┘
```

**设计要点**：该分层使"深度模型 — 量化规则 — 大模型 — 前端"四者完全解耦，可独立替换与单测；
量化内核为纯函数（仅依赖标准库 `math`），可脱离 PyTorch 单独运行验证；同一新闻在 eval 模式下
结果可复现。

## 3. 阶段一：从 TGRFN 提取研判中间量

AI 研判不另起炉灶，而是**复用 TGRFN 互注意力协同检测模块（报告 §2.2.4）已经算出的物理量**，
通过给检测模型增加一个旁路开关 `forward(..., return_internals=True)` 取出。该开关默认关闭，
关闭时返回签名 `(predictions, dist)` 与原训练流程**完全兼容、训练代码零改动**；开启时额外返回
中间量封装 `ModelInternals`：

| 中间量 | 来源（对应报告中的量） | 研判含义 |
|--------|------------------------|----------|
| `probability` | 分类器 `sigmoid` 输出 | 真伪概率（越接近 1 越"真"） |
| `dist_mse` | $\mathrm{MSE}(\text{align}_c,\ \text{align}_g)$，即训练对齐损失 $\mathcal{L}_{dis}$ | 内容表示 ↔ 传播图结构表示的**对齐误差** |
| `rep_scale` | $\tfrac12(\lVert \text{align}_c\rVert^2 + \lVert \text{align}_g\rVert^2)$ 均值 | 表示能量，用于把对齐误差归一化为尺度无关量 |
| `modality_energy` | 前向时缓存的 text/image/time 三模态表示能量 | 各模态对判别的贡献量（呼应多模态模块 §2.2.1 与时序模块 §2.2.2） |
| `num_neighbors` | 参与聚合的传播邻居规模 | 传播规模，用于稀疏/早期判定 |

**为什么直接复用 $\text{align}_c$ 与 $\text{align}_g$**：这两个表示本就是 TGRFN 为
"内容引导传播 / 传播引导内容"的双向互注意力对齐而产生的（报告式 (4)），是模型训练目标的一部分。
直接以它们的差异作为"结构冲突"信号，**不引入任何未经训练的新量**，从而保证研判证据与
检测模型同源、可信。

## 4. 阶段二：量化研判内核（internals.py）

把 5 个原始中间量经一组有明确物理意义的规则，汇聚为结构化研判指标 `JudgmentMetrics`。
逐项给出公式与设计依据。

### 4.1 判别间隔与校准判别（对抗类别不平衡）

设真伪概率为 $p$，训练集正负样本比 $\text{ratio} = \dfrac{\#\text{真新闻}}{\#\text{假新闻}}$，定义**判别间隔**：

$$
\text{margin} = \operatorname{logit}(p) - \log(\text{ratio}) = \log\frac{p}{1-p} - \log(\text{ratio})
$$

$$
\hat{y} =
\begin{cases}
\text{REAL}, & \text{margin} \ge 0\\[2pt]
\text{FAKE}, & \text{margin} < 0
\end{cases}
$$

**原理**：真实判别准则是把后验几率 $\text{odds}=p/(1-p)$ 与样本比 $\text{ratio}$ 比较，
等价于上式比较 $\operatorname{logit}(p)$ 与 $\log(\text{ratio})$。当真假样本不均衡（报告 §3.2 指出该问题）时，
固定 0.5 阈值会系统性偏向多数类；以 $\text{ratio}$ 为基准，相当于**把先验比例从决策中校准掉**，
让"偏离基准的程度"而非"绝对概率"决定结论。$\text{margin}$ 离 0 越远，结论越稳健。

### 4.2 假新闻风险

把判别间隔单调映射到 $[0,1]$ 的假新闻风险：

$$
\text{fake\_probability} = \sigma\big(-\,\text{margin}\cdot s\big),\qquad s=\text{FAKE\_RISK\_SCALE}
$$

$\text{margin}$ 越负（越偏向虚假），风险越接近 1；该值是前端仪表盘与风险分级的主指标。

### 4.3 结构冲突指数（水军操纵 / 异常传播图谱）

$$
\text{structural\_conflict\_index} = \frac{\text{dist\_mse}}{\text{rep\_scale} + \varepsilon},\qquad
\text{alert} = \big[\text{index} \ge \theta_{c}\big]\ (\theta_c=0.5)
$$

**物理含义**：当一条新闻的**内容语义表示**与其**真实传播图结构表示**严重不一致时，说明
"内容本身平平无奇，但它被转发/互动的方式高度不自然"——这正是报告 §1.1、§4 所强调的
**有组织水军账号同步刷量、协同造假**的典型特征。本指标把 TGRFN 关系增强模块（§2.2.3）
所刻画的隐藏协同结构，量化成一个可阈值化的"水军嫌疑度"。除以平均表示能量 `rep_scale`
是为了消除表示尺度影响，得到**跨样本可比**的相对冲突度。

### 4.4 模态注意力贡献（视觉篡改 / 时间爆发 / 文本驱动）

对三模态表示能量做 softmax 归一化，得到和为 1 的注意力占比：

$$
\text{modality\_attention} = \operatorname{softmax}\big(\{e_{text},\,e_{image},\,e_{time}\}\big),\quad
\text{dominant} = \arg\max_k(\cdot)
$$

当某一模态占比超过阈值 $\theta_m=0.6$ 时触发"单模态异常主导"告警，并据主导模态给出定性提示：

- **image 主导** ⇒ 视觉特征注意力异常，提示**配图篡改 / 移花接木 / 合成嫌疑**（建议反向图搜溯源）；
- **time 主导** ⇒ 时间特征注意力异常，呈**爆发后骤减的典型谣言式时间衰减**（呼应时序模块 §2.2.2）；
- **text 主导** ⇒ 判别主要由文本驱动，建议核查文本事实与信源。

### 4.5 传播稀疏判定与置信度修正（早期预警的审慎性）

$$
\text{sparse} = \big[\text{num\_neighbors} < N_0\big]\ (N_0=10)
$$

$$
\text{confidence} = \min\!\Big(1,\ \frac{|\text{margin}|}{3}+0.3\Big);\qquad
\text{若 sparse 则 } \text{confidence} \mathrel{*}= 0.7
$$

**原理**：早期/冷启动阶段（报告 §2.3）传播数据极少，模型易过自信。检测到稀疏即**主动打 7 折**
并提示"持续监测、待传播充分后复判"——宁可保守，也不给过绝对的结论，契合"早发现、早阻断"
但不误伤的防御要求。

### 4.6 量化预分级（不依赖 LLM 的兜底基线）

$$
\text{preliminary\_risk} =
\begin{cases}
\text{SAFE}, & \text{fake\_probability} < 0.35\\
\text{SUSPICIOUS}, & 0.35 \le \text{fake\_probability} < 0.65\\
\text{HIGH\_RISK}, & \text{fake\_probability} \ge 0.65
\end{cases}
$$

若触发结构冲突告警，则整体**上调一档**。该预分级是不依赖大模型的确定性兜底，
同时作为 LLM 的"重要参考"，约束其综合分级不要乱跳。

### 4.7 量化研判指标汇总（JudgmentMetrics）

| 字段 | 类型 | 含义 | 前端用途 |
|------|------|------|----------|
| `fake_probability` | float[0,1] | 校准后假新闻风险 | 风险仪表盘 |
| `model_raw_score` | float[0,1] | 模型原始 sigmoid 输出 | 辅助 |
| `decision_margin` | float | 判别间隔，≥0 判真 / <0 判假 | 辅助/调试 |
| `predicted_label` | REAL/FAKE | 判别标签 | 结论区 |
| `ratio_baseline` | float | 训练集正负样本比基准 | 说明文案 |
| `structural_conflict_index` | float≥0 | 内容↔传播图结构冲突度 | 水军嫌疑警示条 |
| `structural_conflict_alert` | bool | 是否触发结构冲突告警 | 高亮 |
| `modality_attention` | {text,image,time} | 归一化模态贡献（和=1） | 雷达图/条形图 |
| `dominant_modality` / `..._alert` | string/bool | 主导模态及异常告警 | 模态标注 |
| `num_neighbors` / `sparse_propagation` | int/bool | 传播规模与稀疏判定 | 置信度提示 |
| `preliminary_risk_level` | SAFE/SUSPICIOUS/HIGH_RISK | 规则预分级 | 兜底等级 |
| `confidence` | float[0,1] | 量化结论置信度 | 仪表盘副指标 |
| `triggered_rules` | string[] | 命中的可读规则 | 证据兜底 |

## 5. 阶段三：大语言模型解释层

### 5.1 职责边界——"只翻译、不裁决"

LLM 接收阶段二的量化指标（只读 JSON），输出固定字段的结构化中文研判报告。其**职责被三条铁律严格约束**
（写入 System Prompt）：

1. **不得重算、不得推翻**材料中的任何数值（概率、结构冲突、模态贡献均为模型事实）；
2. **不得编造**材料里没有的证据/来源/数据，证据只能引用 `triggered_rules` 与给定指标；
3. **严格输出 JSON**，便于前端直接消费。

这是本系统**反幻觉**的核心机制：大模型强在组织语言、弱在守事实，因此只让它做擅长的事——
把已经算好且可核查的数字，写成专业、分层、可读的报告。由此既获得大模型的表达力，
又从结构上杜绝了报告 §1.2.4 所述的事实性幻觉。

### 5.2 研判规则映射（提示词内置）

提示词把量化指标映射为自然语言研判要点（数值一律以材料为准）：

- 分类与置信度：依据 $\text{margin}$ 与 $\text{fake\_probability}$，越偏离基准越自信；
- 结构冲突：`structural_conflict_alert=true` 时必须在传播分析中点明**水军操纵/异常传播图谱**嫌疑；
- 模态异常：按 `dominant_modality` 指出视觉篡改 / 时间爆发 / 文本驱动；
- 传播稀疏：`sparse_propagation=true` 时必须说明结论基于早期/稀疏数据、置信度已下调、建议持续监测。

### 5.3 输出字段（研判报告契约）

```jsonc
{
  "Risk_Level": "高危|存疑|安全",          // 综合风险等级（核心结论主标）
  "Conclusion": "2~3 句核心结论",
  "Evidence": ["逐条证据，每条挂靠具体数值", "..."],   // ≥4 条
  "Propagation_Analysis": "≥150 字的传播/水军/时间分析段",
  "Recommendation": "分层处置建议（口径 + 具体动作）",
  "_provider": "anthropic|openai|mock|mock(fallback...)",
  "_error": "仅降级时出现"
}
```

报告要求每条证据必须引用具体数值（如 `结构冲突指数 0.93`、`视觉模态占比 78%`、`传播邻居数 6`），
做到专业、可核查、不夸大，并以"模型预测/传播风险研判"口径表述，不写成已坐实的事实核查定论。

### 5.4 工程稳健性（保证服务永远可用且诚实）

- **provider 抽象**：`anthropic` / `openai` / `mock` 三实现同签名，按配置切换厂商；
- **自动降级**：真实调用一旦失败（网络/限流/无密钥/解析异常），自动回退规则版报告，
  并在 `_provider`/`_error` 标注来源与错误摘要——**接口永远返回结构一致的 report**；
- **mock 非占位**：规则版报告同样逐项引用指标，生成内容充实的证据/传播分析/分层建议，
  保证离线、无密钥环境亦可演示完整研判；
- **JSON 兜底解析**：容忍 ```json 代码块包裹或前后多余文本，稳健抽取 JSON 对象。

## 6. 阶段四：服务接口与前端展示

### 6.1 接口契约（metrics 与 report 双段返回）

`POST /api/analyze` 返回两段数据：

```jsonc
{ "metrics": { ...量化指标... }, "report": { ...LLM 研判报告... } }
```

**为什么分离**：`metrics` 是机器可读、可校验的硬数据（前端画图、阈值判断靠它），
`report` 是人可读的软解释（来自 LLM）。二者来源不同、分开返回，使得即便 LLM 不可用，
前端**仅凭 metrics 也能完整展示**。另提供 `with_report=false` 快速模式（只算指标、不调 LLM）用于压测/联调。
配套 `GET /health` 暴露模型/数据/设备/厂商状态，供前端进入页面时探活。

### 6.2 前端研判展示

- **研判等级条**：以 `Risk_Level`（高危/存疑/安全）着色，头部直显**结构冲突指数**与**主导模态**，自适应换行完整呈现；
- **传播拓扑动态演化**：t→t+60 时间轴可拖动回看，渲染关系增强后的传播图谱与隐藏协同结构（呼应研究目标 3 可视化）；
- **详细研判报告页**：风险徽标 + 核心结论 + 关键证据列表 + 传播/结构分析 + 处置建议 + 量化指标快照 + 模态贡献条；
- **一键导出 PDF**：汇总案例信息、模型输出、完整 AI 研判报告与说明（呼应研究目标 4 的研判报告留档）。

## 7. 与 TGRFN 主模型的衔接对照

AI 研判层并非独立发明指标，而是把 TGRFN 已有建模思想**显式化、可解释化、可处置化**：

| TGRFN 建模思想（报告 §2.2 / §4） | 在 AI 研判层的对应体现 |
|----------------------------------|------------------------|
| 互注意力协同检测的内容↔传播双向对齐（§2.2.4） | 结构冲突指数 = $\mathrm{MSE}(\text{align}_c,\text{align}_g)$ 归一化 |
| 关系增强 / 隐藏协同网络挖掘（§2.2.3、研究目标 2） | 结构冲突告警 → 水军操纵 / 异常传播图谱嫌疑 |
| 多模态语义提取（§2.2.1） | 文本/视觉模态注意力贡献与视觉篡改告警 |
| 时序感知传播图构建、时序衰减 $s_o=e^{-g_o}$（§2.2.2） | 时间模态贡献与爆发式时间衰减告警 |
| 早期/冷启动检测（§2.3、研究目标 1） | 传播稀疏判定 → 置信度下调 + 持续监测建议 |
| 类别不平衡评估（§3.2） | $\text{odds}$ 与 $\text{ratio}$ 校准判别（而非 0.5 阈值） |

## 8. 完整数据流（端到端时序）

```text
新闻+传播子图
   │  data_adapter 装配 DataBundle（唯一数据集成点）
   ▼
InferenceEngine.analyze(news_id)          # 进程级单例，避免重复加载权重
   │  model.forward(return_internals=True) → ModelInternals
   ▼
internals.compute_judgment_metrics(...)   # 纯函数 → JudgmentMetrics（量化指标）
   ▼
llm_report.generate_llm_report(metrics)   # LLM 翻译；失败自动降级 mock
   ▼
api.py: { metrics, report, elapsed_ms }   # FastAPI 双段返回
   ▼
前端.html: 研判等级 / 拓扑演化 / 详细报告 / PDF 导出
```

## 9. 关键设计取舍与创新点

1. **可解释优先**：每条研判结论都能回溯到一个具体数值/告警，审核员可逐项复核——这是内容安全场景的底线。
2. **大模型只翻译、不裁决**：以三条铁律 + 只读输入隔绝幻觉，把真伪裁决权牢牢留在 TGRFN，与报告 §1.2.4 的幻觉论述形成闭环。
3. **校准判别对抗不平衡**：以 $\text{odds}$ vs $\text{ratio}$ 取代固定阈值，从判别准则层面消除样本不均带来的系统性偏移。
4. **结构冲突量化水军**：将关系增强模块刻画的隐藏协同结构压缩为一个可阈值化的"水军嫌疑度"，使隐蔽化造假可被显式预警。
5. **稳健降级**：provider 抽象 + mock 兜底 + 数据未挂载 503 明确报错，使系统在 LLM/数据缺位时仍可用且诚实。
6. **解耦可复现**：纯函数内核、数据契约分层、单一集成点、可复现推理，便于单测、交接与替换。
7. **配置驱动**：所有阈值/密钥/路径集中在 `config.py`，换数据集或换厂商不动业务代码。
